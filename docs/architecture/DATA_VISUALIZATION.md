## 数据大小变化

```
PDF文件 (1-5 MB)
    │
    ├─→ [文本提取]
    │   paper_text (50-200 KB) ━━━━━━━━━━━━┓
    │   sections (10-50 KB)                ┃
    │   metadata (~1 KB)                   ┃ 压缩
    │                                      ┃ 约100x
    ├─→ [AI分析]                           ┃
    │   PaperAnalysis (5-10 KB) ━━━━━━━━━━┛
    │   PresentationContent (10-20 KB)
    │
    ├─→ [缓存]
    │   runtime/cache.json (15-30 KB)
    │
    ├─→ [生成]
    │   Markdown (20-40 KB)
    │
    └─→ [输出]
        HTML (50-100 KB)
        PDF (500 KB - 2 MB)
```

## 总结

PaperReader的数据流程特点：

1. **模块化** - 每个阶段职责明确
2. **可追踪** - 所有中间产物可检查
3. **可缓存** - 避免重复计算和成本
4. **可恢复** - 错误可定位和重试
5. **可扩展** - 易于添加新功能

关键优化：
- ✅ 缓存机制 (节省70%成本)
- ✅ 智能分段 (降低API调用)
- ✅ 错误重试 (提高可靠性)
- ✅ 进度反馈 (改善体验)
